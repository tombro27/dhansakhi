# DhanSakhi — Blueprint & Grounded Research

> **IDBI Innovate 2026 · Track: Digital Wealth Management**
> "Build. Integrate. Transform." — an AI Robo-Advisor for IDBI Bank.

DhanSakhi (धन-सखी, *"wealth friend"*) is an **AA-native, explainable, vernacular AI wealth
co-pilot** for IDBI Bank customers. It aggregates a customer's *entire* financial life via the
RBI Account Aggregator framework, builds goal-based plans, gives **explainable** portfolio advice,
talks to the customer in **plain, regional language** (Gemini-powered), and cross-sells the
**IDBI + LIC** product shelf as genuine goal-completion — all inside the trust of a bank.

---

## 1. The problem (specific, Indian, numbered)

India's mass-affluent customer — e.g. *Meena, 38, Pune, an IDBI customer* — has savings scattered
across 4+ institutions (bank FDs, MF folios, demat equity, EPF, an old LIC policy), **no
consolidated view, and no affordable advice**. Quality advisory is reserved for HNIs; everyone else
gets either DIY apps or commission-driven mis-selling. Existing apps either:

- **aggregate but don't advise** (Groww, Zerodha Coin — closed-garden, execution-first), or
- **advise but aren't trusted / are paywalled** (INDmoney/INDwealth ~₹400/mo, lead-gen concerns), or
- **are bank apps that are import-based and HNI-only** (HDFC SmartWealth, ICICI Money Coach).

**Nobody** combines bank trust + AA-native aggregation + *explainable* AI + vernacular advice +
a captive IDBI+LIC product shelf. That intersection is DhanSakhi's wedge.

## 2. Why now (grounded)

- **RBI Account Aggregator is mainstream rails**: ~2.61B accounts enabled, ~252.9M linked users,
  410+ FIUs (Dec 2025); SEBI's Aug-2022 cross-sectoral circular brought NSDL/CDSL in as FIPs, so
  ~81M demat accounts and ~135.6M MF folios are now shareable in real time. AA-facilitated lending
  hit ₹1.47L cr in H1 FY26 (+71% YoY) — production-grade, at scale.
- **IDBI already owns the distribution shelf**: AMFI MF distributor (ARN 0058), broking via IDBI
  Capital Markets & Securities, NPS Point-of-Presence — recommendations route to *real* rails.
- **LIC synergy is structural**: LIC holds ~49.24% of IDBI (Govt ~45.48%); IDBI is LIC's #1
  bancassurance partner. Insurance/annuity cross-sell is a defensible moat no standalone robo has.
  *(Note honestly: a ~60.72% Govt+LIC strategic stake sale was under evaluation in early 2026;
  bancassurance ties are intended to survive it.)*
- **Explainability is now a regulatory tailwind, not just polish**: SEBI's IA framework expects
  algorithm transparency/testing, human oversight, "advice generated by a system" disclosure, and
  5-year suitability records. Explainable AI satisfies the regulator *and* differentiates.
- **Inclusion mandate**: PMJDY ~56 cr accounts, RBI FI-Index 67.0 (Mar-2025); RBI explicitly pushes
  vernacular financial literacy. Vernacular advice is mission-fit for a PSU-heritage bank.

## 3. The golden path (the demo we nail end-to-end)

`Onboard (KYC-lite)` → `AA consent (one-tap, revocable artefact)` → `360° aggregation` →
`AI risk profile` → `Goal: Retirement` → `Explainable recommendation + rebalance` →
`Vernacular advisor chat` → `One contextual IDBI/LIC cross-sell`.

Everything else (more goals, full marketplace, settings) is visible-but-shallow so the app looks
complete without spreading demo depth too thin.

## 4. USP (top 3, defensible)

1. **AA-native, zero-upload 360° aggregation** — one-tap RBI AA consent pulls bank + MF folios +
   demat + EPF + NPS. No CAS/PDF imports (the friction INDmoney/HDFC SmartWealth still carry).
2. **Explainable AI advice** — every recommendation & rebalance ships a Gemini-generated,
   profile-linked, *vernacular* "why this, why now, what changed" rationale. Beats black-box
   curation (Scripbox) and rule-based recs (everyone) **and** maps to SEBI transparency.
3. **Bank+LIC goal-completion cross-sell** — goals map to *real* products: Retirement → NPS (IDBI
   POP) + LIC annuity; Protection → LIC term; Debt sleeve → IDBI FD; Growth → IDBI-distributed MF.
   Framed as filling a goal gap, with transparent reasoning — not push-selling.

## 6. Architecture (Next.js + Vercel + Gemini)

```
Customer (web/mobile)
      │
      ▼
Next.js 16 App Router  ──────────────┐
  • Onboarding / Dashboard / Goals   │  React 19 + Tailwind v4 + Recharts
  • Explainable advice / Advisor     │
      │ (route handlers / API)       │
      ▼                              │
  ┌──────────────┬──────────────┬───────────────┬─────────────────┐
  │ AA Connector │ Recommendation│ Gemini API     │ IDBI/LIC Product │
  │ (simulated)  │ Engine (TS)   │ (advisor +     │ Catalog          │
  │ FIP→FIU,     │ risk, alloc,  │  explainability)│ (FD/MF/NPS/LIC)  │
  │ consent      │ goals, cross- │  with no-key    │                  │
  │ artefact     │ sell mapping  │  fallback       │                  │
  └──────────────┴──────────────┴───────────────┴─────────────────┘
      │
      ▼
  Synthetic data store (no real PII)  →  POC: IDBI core banking + AA FIU + RIA review
```

- **AA Connector (simulated)** mirrors the real FIP→AA→FIU consent flow + consent artefact JSON.
- **Recommendation Engine** is deterministic TypeScript (risk scoring, target allocation,
  rebalancing deltas, SIP-to-goal future-value math, cross-sell mapping). No AI needed → reliable.
- **Google Gemini** powers the *conversational advisor* and the *natural-language explainability*; it
  degrades gracefully to curated responses when `GEMINI_API_KEY` is absent (deploy always works).
- **Compliance by design**: advice = "decision-support + education surfacing IDBI-RIA-vetted
  recommendations", human-in-the-loop, suitability logged, synthetic data only.

## 7. Compliance posture (designed-in, honest)

- **SEBI (Investment Advisers) Regs 2013** — DhanSakhi is positioned as explainable
  aggregation + decision-support + education that surfaces **IDBI-RIA-reviewed** recommendations
  with disclaimers and human-in-the-loop oversight; advice and execution kept architecturally
  separate (SEBI EOP framework). Suitability/risk-profile records retained.
- **RBI Account Aggregator** — consent-based, revocable, data-blind AA; no scraping.
- **DPDP Act 2023** (Rules notified 13-Nov-2025) — explicit consent, purpose limitation, data
  minimization; demo uses synthetic, no-real-PII data.

## 8. Cross-sell ROI model (benchmark-anchored, labeled as projection)

Industry: generic cross-sell converts **1–3%**; AI next-best-offer personalization **8–18%**
(~6× lift); revenue uplift **25–40%**; payback **18–36 months**. Applied to IDBI's active base with
AA-driven full-portfolio visibility → a credible incremental-revenue story (see deck slide 9/11).

## 9. Benchmarking (prototype, honest)

- AA aggregation simulated < 2s for a 6-institution portfolio.
- Explainability: every recommendation carries a profile-linked rationale (100% coverage).
- Advisor latency: Gemini responses ~1–3s; deterministic fallback < 50ms.
- Vernacular PoC evidence (literature): +41% task completion, +86% session length vs English-only.

## 10. Roadmap

Pilot (sandbox) → live AA FIU integration → IDBI core-banking + LIC catalog APIs → RIA review
workflow → production rollout to IDBI's mass-affluent base, vernacular-first.
